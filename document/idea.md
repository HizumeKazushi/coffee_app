# **コーヒー愛好家向けアプリ「COFFEE RECIPE HUB」要件定義書**

## **1\. プロジェクト概要**

### **1.1 目的**

コーヒー抽出の「再現性」を高め、ユーザー同士で最適な「レシピ」と「豆情報」を共有することで、日々のコーヒー体験を向上させる。

### **1.2 コンセプト**

「記録(Record)・再現(Reproduce)・共有(Share)」  
単なる日記ではなく、美味しい一杯に辿り着くための「ツール」としての価値を提供する。

## **2\. ユーザー機能要件**

### **2.1 豆管理（Bean Management）**

* **登録機能**: 焙煎所名、豆の名前、産地、焙煎度、精製方法、焙煎日の記録。  
* **エイジング通知**: 焙煎日から算出した「飲み頃（ピーク）」をプッシュ通知。  
* **在庫管理**: 残りg数の管理（抽出ごとに自動減算）。

### **2.2 レシピ作成・編集（Recipe Editor）**

* **基本設定**: 器具（ドリッパー等）、粉量、総注湯量、お湯温度、挽き目の設定。  
* **タイムライン設定**:  
  * 「○秒目に○g注ぐ」というステップを複数登録。  
  * 各ステップにラベル（蒸らし、1投目など）を付与。  
* **公式レシピインポート**: 焙煎所が発行するQRコードからのレシピ取り込み。

### **2.3 抽出セッション（Live Brewing）**

* **タイマー機能**: 0.1秒単位の抽出タイマー。  
* **リアルタイムナビゲーション**: 現在の注湯量と次のアクションを視覚的にガイド。  
* **ログ保存**: 抽出完了後、実際の抽出結果と「味の感想（5段階評価＋メモ）」を紐付けて保存。

### **2.4 コミュニティ・共有（Community）**

* **レシピ公開**: 作成したレシピのタイムラインを他のユーザーに公開。  
* **Brewed it\!（やってみた）**: 他者のレシピを試した際のフィードバック投稿。  
* **フォロー・フォロワー**: 特定のユーザーや焙煎所の更新を追跡。

### **2.5 AI味わいアドバイザー（Advanced）**

* **改善提案**: ユーザーの感想（例：「苦すぎた」）に対し、レシピ（温度、挽き目、時間）の修正案をAIが提示。

## **3\. 非機能要件**

### **3.1 ユーザビリティ（UI/UX）**

* **片手操作**: 抽出準備中の手が塞がっている状況を想定し、主要操作を片手で完結。  
* **ダークモード対応**: 早朝や夜間の落ち着いた時間帯での利用に配慮。  
* **オフライン対応**: 通信環境の悪い場所でも基本的なタイマーとログ保存が可能。

### **3.2 パフォーマンス**

* **リアルタイム性**: タイマーのラグを最小限（100ms以内）に抑える。  
* **高速起動**: 抽出をすぐに始められるよう、アプリ起動からタイマー開始までの遷移を3タップ以内にする。

## **4\. データ構造（主要エンティティ）**

| エンティティ | 主要属性 |
| :---- | :---- |
| **User** | ID, ユーザー名, 好みの味の傾向 |
| **Bean** | ID, 豆名, 焙煎所ID, 焙煎日, プロファイル(酸味/苦味等) |
| **Recipe** | ID, タイトル, 器具ID, ステップ(Array), 作成者ID |
| **BrewLog** | ID, レシピID, 豆ID, 実施日時, 味の評価, 改善メモ |

## **5\. 将来の拡張性（ロードマップ）**

* **ハードウェア連携**: Bluetooth対応スマートスケール（Acaia, Hiroia等）とのリアルタイム同期。  
* **EC連携**: 評価の高い豆をアプリ内から直接購入できるロースタリーマーケット。  
* **画像解析**: 豆のパッケージを撮影するだけで情報を自動入力するOCR機能。
